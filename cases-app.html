Cases App

Next.js + SuiteCRM — beginner guide to CRUD, routing, and the V8 API

What This App Does

A small web app that manages Cases in SuiteCRM. You can list, add, edit, and delete cases from a browser page.

Data is stored in SuiteCRM — not in the Next.js app.

Built for beginners: the UI talks to your API routes, and those routes talk to the SuiteCRM V8 API.

The browser never calls SuiteCRM directly. It only calls /api/cases.

How Routing Works

Next.js App Router — folders + special file names

Next.js does not connect URLs to component names like CasesPage. It uses folders + special file names under app/.

Folder path under app/ + file named page.tsx = URL route

File on diskURL in browser
app/page.tsx/
app/cases/page.tsx/cases
app/about/page.tsx/about

Does /cases Know CasesPage?

No. The function name is for developers only.

  1. Browser requests /cases
  2. Next.js maps segment cases → folder app/cases/
  3. Next.js loads page.tsx inside that folder (required filename)
  4. Next.js renders the export default function from that file
  5. Output is wrapped inside app/layout.tsx
// app/cases/page.tsx export default function CasesPage() { // export default = use this component for this route // CasesPage = optional label; rename without changing URL }

What Runs When You Open /cases

http://localhost:3000/cases │ ▼ Next.js router matches URL segment "cases" │ ▼ Finds app/cases/page.tsx │ ▼ Renders default export (CasesPage component) │ ▼ Wraps it in app/layout.tsx as {children} │ ▼ Final HTML sent to browser
// app/layout.tsx wraps every page export default function RootLayout({ children }) { return ( <html lang="en"> <body>{children}</body> {/* CasesPage output goes here */} </html> ); }

Pages vs API Routes

KindExample URLFileRuns where
Page (UI)/casesapp/cases/page.tsxBrowser
API/api/casesapp/api/cases/route.tsServer only

The cases page calls the cases API with fetch("/api/cases"). Separate routes, separate files.

Mental model: URL path = folder path under app/ Page UI = default export in page.tsx API handler = GET/POST/PATCH/DELETE in route.ts Component name = for humans only; router ignores it

Flow 1: Open /cases (Load List)

Trigger: user opens page or navigates to /cases

User opens /cases → Next.js loads app/cases/page.tsx → CasesPage mounts → useEffect → loadCases() → fetch("GET /api/cases") → app/api/cases/route.ts → GET() → lib/suitecrm.ts → listCases() ├─ getToken() → POST SuiteCRM /Api/access_token └─ crmFetch() → GET SuiteCRM /Api/V8/module/Cases?... → Each record → toCase() → JSON array to browser → setCases(data) → list renders State: loading true→false | cases filled | error if fail

Flow 2: Add a New Case

Trigger: user fills form and clicks Add

User clicks Add → handleSubmit() → validate title → fetch("POST /api/cases", { title, description, status }) → app/api/cases/route.ts → POST() → createCase(body) → toCrmFields() (title→name, status→Open_New) → crmFetch("POST /Api/V8/module", { type: "Cases", attributes }) → SuiteCRM saves row → toCase() → new id → Browser updates list → clearForm()

Flow 3: Edit a Case

Click Edit

  • setEditingId(case.id)
  • Fill form from that case
  • Button shows "Update case"

Click Save

  • fetch("PATCH /api/cases/{id}")
  • [id]/route.ts → PATCH
  • updateCase(id, body)
  • Replace row in list → clearForm()

Flow 4: Delete a Case

Trigger: user clicks Delete on a list row

User clicks Delete → handleDelete(id) → fetch("DELETE /api/cases/{id}") → app/api/cases/[id]/route.ts → DELETE() → deleteCase(id) → crmFetch("DELETE /Api/V8/module/Cases/{id}") → Browser: setCases(cases.filter(...)) — remove from list → If editing same case: clearForm() too

  • useState, useEffect
  • onClick, onChange, onSubmit
